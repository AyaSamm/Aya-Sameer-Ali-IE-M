What are the limitations of CSS?

  • Lack of Variables: CSS lacks built-in support for variables, resulting in code duplication and increased maintenance effort.
  • Global Scope: CSS operates in a global scope by default, leading to style conflicts and unintended side effects.
  • Limited Dynamic Styling: CSS is primarily designed for static styling and has limited capabilities for handling dynamic changes.
  • Limited Layout Control: While CSS has advanced layout capabilities, achieving complex layouts can still be challenging, especially for vertical centering and equal-height columns.
  • Browser Compatibility: Different browsers may interpret and render CSS rules differently, causing inconsistencies in appearance.
  • Lack of Calculation and Expressiveness: CSS has limited mathematical and logical capabilities, restricting flexibility and expressiveness.
  • Print and Page Control: CSS provides basic support for print stylesheets but lacks fine-grained page layout control.
  • Performance Impact: CSS performance can be affected by large and complex stylesheets, leading to slower rendering and increased page load times.

How many ways can CSS be integrated as a web page?

  • Internal Style Sheet: Defined within an HTML page. Requires the use of the <style> element within the <head> section of the HTML document.
  • External Style Sheet: Exists as a separate file with a ".css" extension. Linked to an HTML document using the <link> element. Allows you to change the styling of an entire website by modifying a single CSS file.
  • Inline CSS: Has the highest priority among the three CSS methods (external, internal, and inline). Applied directly to HTML elements as attributes. Overrides styles defined in external or internal CSS, but it's recommended to use it sparingly.

In summary, these three methods offer different ways to apply CSS styles to web content, each with its own use cases and priorities.
